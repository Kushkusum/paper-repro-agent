Enhanced Qwen-VL 7B Model via Instruction Finetuning on Chinese Medical Dataset
Jianping Luo, Cong Tan, Hanyi Yu, Haitao Yu

Abstract: This study focuses on the development of a medical-specific question-answering model
leveraging the Qwen-VL 7B model, an advanced Large Vision-Language Model (LVLM), to enhance the
understanding and generation of medical texts. Results showed that the Qwen-VL-Medical model
achieved a Rouge-1 score of 0.6147, indicating its potential for medical applications.

II. METHODS
A. Qwen-VL-Medical

We trained the Qwen-VL 7B model, focused on processing medical imaging data. The model's total
size reaches 9.07 GB (Vision Encoder 1.9B + VL Adapter 0.08B + LLM 7.7B = 9.6B parameters total).
We integrated a dataset containing X-ray image questions and answers, set the maximum sequence
length to 1800, performed preprocessing, adjusted the batch size to 2 (later text says 642 in the
training section), used gradient accumulation of 8x, learning rate 1e-5, AdamW optimizer, cosine
learning-rate schedule. After training, the model reached 0.6147 on the Rouge-1 evaluation index,
with training loss reduced to about 0.75. We named this model Qwen-VL-Medical.

B. Data Process

The dataset was sourced from the National Library of Medicine's (NLM) Open-i platform, an
open-access biomedical image search engine. After cleaning, the dataset comprises 7,000 X-ray
images, divided into training/testing sets via five-fold cross-validation. Questions and answers
for each image were generated using the GPT-4 model based on each image's diagnostic report. This
custom-built question/answer dataset (images + GPT-4-generated QA pairs) is not publicly released
-- there is no dataset link, DOI, or repository ID given anywhere in the paper.

C. Pretrained VL-Model

Qwen-VL is a Large Vision-Language Model whose foundation is a large language model initialized
from pretrained Qwen-7B weights, plus a ViT-based visual encoder and a position-aware
vision-language adapter. Qwen-VL's own pretraining (not redone by this paper, used as a frozen
starting point) involved 5 billion image-text pairs (reduced to 1.4 billion after cleaning),
50,000 pretraining steps with batch size 30720, followed by multi-task pretraining and instruction
fine-tuning stages, originally requiring large-scale GPU compute clusters.

D. Training (fine-tuning stage performed by this paper)

QLoRA is used: an efficient fine-tuning method allowing fine-tuning of models with up to 6.5
billion parameters on a single 48GB GPU, via a frozen int4-quantized pretrained language model
with gradients backpropagated to low-rank adapters (LoRA), using 4-bit NormalFloat (NF4)
quantization, double quantization, and paging optimizers.

ROUGE is the evaluation metric used (ROUGE-1, ROUGE-2, ROUGE-L), measuring n-gram/sequence overlap
between generated and reference answers.

III. RESULTS AND ANALYSIS

Comparison of three vision-question-answering models on the held-out test set of the custom
medical dataset:

Models      Rouge-1   Rouge-2   Rouge-L
LLava       0.5801    0.3415    0.5521
VisualGlm   0.5816    0.3440    0.5559
Qwen-VL     0.6147    0.3535    0.5796

Qwen-VL outperformed LLava and VisualGlm across all ROUGE metrics on this held-out medical VQA
test set.

IV. CONCLUSION

The study fine-tuned the 9.6B-parameter Qwen-VL 7B model (via QLoRA on a 48GB GPU) on a
custom-built, unreleased 7,000-image medical X-ray VQA dataset (images from NLM Open-i, QA pairs
generated by GPT-4), achieving Rouge-1 = 0.6147 after fine-tuning.
